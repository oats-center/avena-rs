Rust + NATS + Parquet + Svelte

Avena LabJack data capture from sensor input to exportable files.

This repo streams LabJack T7 analog input samples into NATS, archives those samples as timestamped parquet files, serves CSV exports over a WebSocket endpoint, and provides a Svelte dashboard for configuration, live plots, and export requests.

Start locally See the data flow

Quickstart

Run the pipeline pieces

Build Rust binaries

cd rust-ljm
cargo build --release

Start managed processes

./streamerctl.sh start
./archiverctl.sh start
./exporterctl.sh start

Run the dashboard

cd webapp
pnpm install
pnpm run dev -- --host 0.0.0.0

Run validation

python3 ljstreamM_data/validate_signals.py run-suite \
  --config ljstreamM_data/validation_suite.json

System overview

How data moves through the repo

Sensors Analog input AIN channels on LabJack T7
->
streamer FlatBuffer scans Per-channel NATS subjects
->
NATS JetStream + KV Config and sample transport
->
archiver Parquet files timestamp_unix_ns, value
->
exporter CSV stream WebSocket /export
->
Svelte webapp Plots + downloads Browser UI for operators

Runtime paths

  • streamer runs on the edge host with Ethernet access to the LabJack.
  • archiver runs wherever parquet should be stored.
  • exporter must run on the same host as the parquet directory it serves.
  • webapp runs in the browser and talks to NATS WebSocket plus exporter WebSocket.

NATS subject shape

avenabox.001.data.ch11
└prefix └asset └kind └channel

The streamer creates or updates the JetStream stream configured by nats_stream and publishes every enabled channel under {nats_subject}.{asset_number:03}.data.ch{channel:02}.

Rust binaries

What each binary does

streamer

Opens the LabJack T7 through a strict Ethernet IP path, verifies device type and optional serial number, starts LJM streaming, and publishes one FlatBuffer scan per channel per read batch.

Inputs
NATS credentials, KV config, LABJACK_IP, optional LABJACK_SERIAL.
Output
NATS payloads containing first_sample_unix_ns, sample_interval_ns, actual_scan_rate_hz, sequence, and values.
Command
cd rust-ljm && ./streamerctl.sh restart

archiver

Loads the same KV config, subscribes to each enabled channel, decodes FlatBuffer scans, expands timestamps for every sample, and writes raw channel values into per-asset, per-day, per-channel parquet files.

Inputs
NATS subjects such as avenabox.001.data.ch11 and calibration specs from KV config.
Output
parquet/asset001/2026-04-22/ch11/part-0001.parquet with parquet metadata carrying calibration JSON.
Command
cd rust-ljm && ./archiverctl.sh start

exporter

Serves a WebSocket endpoint at /export. A browser sends an asset, channels, time range, and format. The exporter scans parquet files, applies calibration in memory, and streams CSV chunks back.

Inputs
Parquet directory plus WebSocket JSON request.
Output
Text metadata frame, binary CSV frames, summary frame, then completion frame.
Command
cd rust-ljm && ./exporterctl.sh start

subscriber

Diagnostic listener for NATS sample traffic. It subscribes to an asset wildcard, decodes FlatBuffer scans, and appends per-channel CSV files.

Inputs
NATS_SUBJECT, ASSET_NUMBER, NATS_CREDS_FILE.
Output
outputs/labjack_001_ch11.csv with sequence,timestamp,raw_value.
Command
cd rust-ljm && cargo run --bin subscriber

Why the config is split this way

KV config is runtime device intent.

The LabJack name, asset number, enabled channels, scan rate, and calibration settings live in NATS KV so the webapp can update them without editing files on the edge host. The streamer and archiver both watch the same source of truth.

Env JSON is host-local process setup.

Credentials, LabJack IP, exporter bind address, and parquet path are machine-specific. Keeping them in *.env.json avoids storing deployment details in the shared KV payload.

Strict LABJACK_IP prevents wrong-device capture.

The streamer intentionally opens the T7 by direct Ethernet IP and verifies the optional serial number. That removes USB fallback and subnet discovery from production capture, where silent device selection mistakes are worse than a hard startup failure.

Calibration is archived with the data.

The archiver writes raw values to parquet and stores calibration metadata on each file. Export time can then reproduce calibrated CSV values using the calibration that was active when the file was written.

Payload examples

Input and output shapes

FlatBuffer scan

table Scan {
  first_sample_unix_ns: ulong;
  sample_interval_ns: ulong;
  actual_scan_rate_hz: double;
  sequence: ulong;
  values: [double];
}

Parquet row

timestamp_unix_ns,value
1776890700000075000,2.1363790035
1776890700000275000,1.5884703398

Export CSV

timestamp,channel,raw_value,calibrated_value,calibration_id
2026-04-22T20:45:00.000075+00:00,ch11,2.136379,2.136379,identity

FlatBuffer field meanings

Field Meaning Reason it exists
first_sample_unix_ns Unix timestamp, in nanoseconds, for values[0]. Gives every batch an absolute time anchor so archive files, plots, and exports do not depend on browser or subscriber receive time.
sample_interval_ns Nanoseconds between consecutive samples in values. Avoids repeating one timestamp per sample on the wire while still allowing exact timestamp reconstruction for every row.
actual_scan_rate_hz The scan rate returned by LJM_eStreamStart. LabJack may return an actual achievable rate. Carrying it in the payload makes downstream timing diagnostics and validation explicit.
sequence Monotonic batch counter for the current streamer run. Consumers can detect gaps, resets, duplicate batches, or streamer restarts without inspecting timestamps alone.
values Raw voltage samples for one channel in this batch. The streamer publishes one FlatBuffer per channel, which keeps NATS subjects channel-addressable and lets the archiver write independent channel files.

Why FlatBuffers and per-channel messages

Binary payloads keep high-rate streams smaller.

FlatBuffers avoid JSON parsing overhead and preserve numeric types directly for Rust and TypeScript consumers.

One batch timestamp plus interval is compact.

A 5 kHz channel does not need 5,000 full timestamps per second on NATS. The archiver reconstructs sample timestamps as first_sample_unix_ns + index * sample_interval_ns.

Per-channel subjects simplify storage and UI subscriptions.

The webapp can subscribe only to channels it is plotting, and the archiver can rotate parquet files per asset, date, and channel.

Raw-first storage preserves future options.

Parquet stores raw values, while exporter output includes both raw and calibrated values. That keeps archival data immutable even if calibration formulas improve later.

Svelte webapp

What the browser UI does

Configuration dashboard

The login page connects to NATS over WebSocket using a creds file. The LabJack page lists labjackd.config.* keys from the avenabox bucket, edits configs, writes updates back to KV, and can save reusable calibration.* presets.

Live plotting and exports

Plot pages subscribe to the same per-channel NATS sample subjects as the archiver, decode the generated TypeScript FlatBuffer schema, and draw source-timestamped data on canvas. Export requests go to the Rust exporter WebSocket and produce CSV files from archived parquet.

Validation

How signal validation is done

Suite command

python3 ljstreamM_data/validate_signals.py run-suite \
  --config ljstreamM_data/validation_suite.json

The default suite compares checked-in LJStreamM references with Avena CSV exports for 10 Hz, 100 Hz, 1 kHz, and 5 kHz captures. The suite emits suite_summary.json, suite_summary.csv, per-case summaries, aligned samples, and plots.

Metrics

  • Timestamp-relative alignment with cross-correlation lag estimate.
  • Absolute error: RMSE, MAE, max, p50, p95, p99.
  • Relative error: mean, p50, p95, p99.
  • Drift: lag slope and amplitude-ratio slope over rolling windows.
  • Pass/fail: configurable JSON thresholds per suite or per case.
